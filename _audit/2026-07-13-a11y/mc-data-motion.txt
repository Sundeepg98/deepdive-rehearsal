   The landmine IS still armed in the LIGHT DOM: styles.css:260 `*{animation:none!important}` strips
   animation-NAME, so a "forwards" fill can never apply. body survives only because its base state is
   opacity:1 + fill "backwards" (styles.css:207). Put opacity:0 + "forwards" back and the app is blank again.

   name — so inside a shadow root the animation still RUNS (in .01ms) and a "forwards" fill still lands.
   => the two halves of this app have OPPOSITE reduced-motion semantics. See mc-01b-landmine.mjs.
