## The three fixes

### 1. Grade at the reveal moment (audit #4, borderline P1) — commit `551790e`
- **Root cause:** the in-pane judge row (`#jm/#js/#jg`, Missed/Shaky/Solid) rendered ONLY at the full
  judgment point (`drawCard` stage >= maxStage) — i.e. after clicking through the ENTIRE "Interviewer
  pushes further" chain. Every probe carries 2–3 follow-ups, so a cold first-timer who revealed the
  answer and moved on never saw the self-grade at all, silently forfeiting the spaced-repetition spine
  the whole app is built on (the Resume hero, the "still shaky" chips, the weakness view — all fed by
  this grade). Grading was not *ignored*; it was **unreachable** without exhausting the chain.
- **Fix (`drill/logic.js`):** render the in-pane judge row the instant the answer is on screen
  (stage >= 1), with "Interviewer pushes further" kept ABOVE it as the optional deepen path — quick-
  grade if confident, or push further for the full test then grade at the boundary. The SAME
  `#jm/#js/#jg` control and the SAME `judge()`: one decision engine surfaced earlier, never a second.
  The must-hit checklist stays at the judgment point (it derives from the senior tell, revealed there),
  so a reveal grade is an honest gut-feel call and records `cov{0,0}` ("coverage not assessed") —
  `_mhp/_cov` are cleared so the previous card's checklist can't bleed into it.
- **DELIBERATE BEHAVIOR CHANGE (flagged, ratified):** this softens the documented "quiet mid-read,
  armed at the judgment" decision — but ONLY for the in-pane row. `_judgeOn` is **UNTOUCHED**, so the
  DOCK's armed-key legend stays quiet mid-read and arms only at maxStage, exactly as shipped. The
  in-pane row is the discoverable surface; the sidebar dock is untouched (the parallel dock wave's
  boundary is preserved).
- **Ratified conditions (team-lead, all covered + tested):**
  1. the row is present at reveal AND persists through the deepen path to maxStage; a reveal-grade
     merges by content-id, latest-wins (grade Missed at reveal → re-drill → grade Solid → record holds
     Solid, ONE entry). grade-advances flow unchanged (the existing merge-by-content-id semantics).
  2. a reveal grade records `cov{0,0}`, distinguishable from a chain-grade — **do not "fix" this into
     fake coverage.** It means "coverage not assessed", which is the honest state when the checklist
     (the senior-tell-derived aid) has not been shown yet.
  3. the in-pane-vs-dock divergence is DELIBERATE and TESTED both sides: the dock is quiet at reveal
     (a load-bearing boundary guard — goes red if anyone re-arms `_judgeOn`/`atJudgment` early) and
     arms at maxStage.
  4. "push further" is a first-class amber-gradient alternative beside the quiet grade row, not a
     footnote.
- **Watched RED** (`test/grade_reveal.cjs`, new): against the pre-fix deliverable the row is absent at
  reveal, so presence, the two-sided shadow hit-test reachability, the real-`page.mouse.click` grade,
  the maxStage-persistence and the merge all fail. Carries a live PLANT (hide the row → the hit-test
  stops finding the button) and a hardening commit (below) after a load flake.

### 2. Cold-open value prop + de-jargoned Start CTA (audit #13a) — commit `1ef9f62`
- **Root cause:** the first minute under-stated identity — "system design" appeared NOWHERE above the
  fold, and the home led with undefined jargon ("Drill the probes"). The linked-in candidate spent 60
  seconds deciding what this even is.
- **Fix (`home-view.js` + `styles.css`):** one quiet value-prop lead (`.hm-lead`) with the key phrase
  in ink and the rest muted, shown to COLD users only (`!Panels.engaged()`) so it never taxes a
  returning user's one-line-of-state / one-decision home. De-jargon the cold Start CTA description.
- **VR:** `home-light` + `home-dark` baselines regenerated — the lead line shifts the cold home's
  content down. Confirmed via `home-light.diff.png` as a **pure vertical shift + the two new copy
  lines**, no layout break. Surgical: only the 2 home `sha256` entries changed in `manifest.json`; a
  noise re-capture of `room-architecture-apis` (an untouched `#state-machine/walk` topic pane) was
  reverted, and the other 14 baselines are byte-untouched. VR re-verify: worst = 0 px on home-light.
- **Watched RED** (`test/cold_open.cjs`, new): pre-fix the `.hm-lead` element does not exist. Asserts
  the cold home names "system design" ABOVE THE FOLD, the Start CTA is de-jargoned, and the lead is
  GONE once engaged (the gating). Live plant (hide the lead → the above-fold value prop is gone).
  Guards STRUCTURE; the words are the operator's to review.

### 3. Whole-system search prompts route to component topics (audit #13b) — commit `3e234f9`
- **Root cause:** a candidate types the prompt they actually rehearse — "design twitter", "url
  shortener", "search autocomplete", "design instagram" — and search returned a bare "No results
  found". This trainer teaches the COMPONENTS such systems are built from (caching, sharding, real-time
  delivery…), not the whole-system prompts, so a blank dead-end on their exact question reads as
  "doesn't have my material" when the parts are all here.
- **Fix (`search-overlay.js`):** a curated whole-system-prompt map (14 prompts) routes a matched miss
  to the real component topics that compose it. HONEST by construction — the section fires ONLY on a
  genuine 0-hit miss (so it can never mask a real result), and every chip is filtered to a topic the
  registry actually holds (never a faked result, never a dead link). The typo-tolerant "Did you mean"
  path is unchanged for non-system misses.
- **Watched RED** (`test/search_deadend.cjs`, new): pre-fix the `[data-sys-topic]` chips do not exist.
  Asserts each audit prompt shows an honest miss AND component chips that are all registry-real, a
  real hit-tested click routes to the topic, and two plants (a gibberish miss + a real component hit)
  both show ZERO chips so the section can't degrade into fire-for-everything.

### Test hardening — commit `7196f2c`
`grade_reveal`'s functional grade-at-reveal assertion flaked once under heavy gate load (the merge
section read the Missed button's box before the reveal row had painted, so the real click hit
nothing). Hardened: wait for the grade row to render after the reveal click, and poll the record until
the grade lands instead of a single post-settle read. Discriminators unchanged (still RED pre-fix);
3× stable green after. (Separately: my first `search-overlay.js` edit tripped `ascii_guard` with a
literal em-dash/middot — caught by the gate, fixed to `—`/`·` escapes, re-gated PASS.)

## Open coherence question (POST-merge, per ratification condition 3 — NOT injected into any wave)

The in-pane grade row now appears at reveal while the dock's armed-key legend stays maxStage-gated.
That divergence is deliberate and tested, but it raises a design question worth settling AFTER this and
the dock wave land: **should the dock's "judgment point" follow the in-pane row (arm at reveal too)?**

keyboard user who can now grade at reveal gets no dock cue that the keys are live (mitigated: the
in-pane buttons carry their own `[1]/[2]/[3]` hints, so the affordance is self-documenting). Flagged
for a post-merge decision; do not fold into the dock wave mid-flight.
