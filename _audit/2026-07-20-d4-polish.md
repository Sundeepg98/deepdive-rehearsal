## The design contract (the mobile wave inherits this)

**"Dock identity = quiet flat panel; accent belongs to the kicker only; exactly ONE loud primary per
surface — the Mock CTA."**

Realized **theme-aware**, because the collapse only existed in dark:
- **Dark** actively flattens the dock (it was an accent-wash card twinning the accent-bordered Mock
  CTA). Bg → `--panel` (flat recessed), border → `--bd` (neutral), CTA label → `--ink`. Accent
  survives only on the kicker.
- **Light** is left byte-identical: the Mock CTA there is a filled gradient (loud) and the washed dock
  is already subordinate, so no flattening is needed. The contract's principle ("one loud primary")
  holds in both; only the mechanism differs.

Confirmed by team-lead as what D5 inherits; its quiet token set (`--card`/`--bd`/`--accink`) is
compatible, and the merge-down handles final token alignment (board task #7).

### Commit 1 — `384d46a` (design cluster)

#### 1. Dark dock / Mock CTA collapse to one tier (audit #3, P1)
- **Root cause:** `.dock` painted `background:var(--accbg)` + `border:1px solid var(--acc-a32)` +
  accent text (styles.css:369). In dark, `--accbg` = `--topic-wash` (a dark tint) and the Mock CTA
  (`html[data-theme="dark"] .mockbtn`, styles.css:519) is a tinted fill + `--topic-solid` border +
  `--topic-ink` text — the SAME family. Live before-measure (data-storage dark): dock bg
  `rgb(27,33,43)` + accent-`/0.32` border; Mock bg `srgb(0.177,0.196,0.274)` + `rgb(125,166,243)`
  border. Two accent-bordered dark cards, stacked. (In light the Mock CTA is a `linear-gradient` fill —
  no collapse.)
- **Fix:** introduce `--dock-bg` / `--dock-bd`. Light resolves them to `--accbg` / `--acc-a32`
  (byte-identical); dark to `--panel` / `--bd` (flat recessed panel). Dark `.nd-go` label → `--ink`
  so accent belongs to the kicker only. Live after-measure (dark): dock bg
  `linear-gradient(#1A1921→#151419)` = `--panel`, border `#37333F` = `--bd`, CTA `#ECEAE4` = `--ink`.
- **Contrast — NOT degraded, IMPROVED:** the dock's MICRO tier still renders the armed grade legend,
  whose AA contrast the audit put at 5.11–6.81:1. Measured on the flattened panel across all 6 rooms
  in dark: **7.02–7.57:1** (higher, because `--panel` is darker than every `--topic-wash`, so the
  light armed-legend glyphs gain contrast).
- **Watched RED** (`test/dock_contrast.cjs`, new): decodes the armed legend's glyphs (Grade / labels /
  digits) against the ACTUAL painted gradient panel. Carries a per-run self-test — a planted light
  dock bg drops the accent digit to **2.19:1** (< the 5.0 floor), or it aborts — so the metric is
  proven to discriminate every run.

#### 2. `--accbg` double-duty (audit #24)
- **Root cause:** the dock background (styles.css:369) and the active desktop seg tab
  (`.sidebar .seg button.on`, styles.css:590) both painted `var(--accbg)` — one token, two semantics,
  adjacent.
- **Fix:** the dock now owns `--dock-bg`; the active tab keeps `--accbg`. Two tokens: visually
  distinct in dark (`--panel` vs `--topic-wash`), byte-identical color in light but semantically
  separated (and the dock's rounded full hairline vs the tab's inset edge-bar already distinguish them
  there). Resolved by the same token split as #3.

#### 3. Seg pip orphaned on the sidebar nav (audit #23)
- **Root cause:** `.seg button.flow-pip:not(.on)::before` tops the pip at `space-4` (styles.css:326) —
  correct for the horizontal strip's short centred button, but on the vertical sidebar nav the label
  row sits at **16.5px** (measured: padding-top space-8 + half a 13px line) while the pip centred at
  **7px** — floating in the top-left gutter, detached.
- **Fix:** `@media(min-width:920px){ .sidebar .seg button.flow-pip:not(.on)::before{top:var(--space-14)} }`
  — pip centre now **17px**, on the label row, as a leading marker in the gutter. Desktop-only, so the
  horizontal strip keeps its base dot (mobile `m-walk-*` baselines are byte-untouched — verified).
  Density-safe (a space token tracks the padding it aligns to).

### Commit 2 — `269896e` (flow-spine a11y)

#### 4. `n` (NextUp) key missing from the Shortcuts overlay (audit #10)
- **Root cause:** `n` is a shipped W2 key (the dock go button wears `aria-keyshortcuts="N"`; shell.js
  handles it globally) but `keyboard-overlay.js` KBD_HTML's "Anywhere" section never listed it.
- **Fix:** one additive KBD row — `N — Go to your next step — the drill, whiteboard or mock the app
  points you at`.
- **Watched RED** (`test/flow_a11y.cjs` #10): pre-fix, no single-`N` row among 23 (absent).

#### 5. Dock CTA / armed legend never announced (audit #18)
- **Root cause:** `#ndock`'s CTA swap and armed-legend swap reach no live region.
- **Fix:** `dockAnnounce()` — a DEDICATED polite region announces the meso/macro CTA when it appears
  (de-duped on the last message). The MICRO armed legend is NEVER passed to it.
- **Why a dedicated region, not the audit's literal `role="status"` on `#ndock` (deviation, flagged):**
  (a) `role="status"` on `#ndock` would announce the armed legend TOO — and the brief's own parenthetical
  bans that ("a judgment point must not shout over the answer text"). (b) `flowDock` recomputes on the
  same `drillgraded` microtask that fires the drill's debrief readout through `ViewManager.announce`;
  sharing that region would collapse them (30ms cancel-pending) and the CTA would clobber the more
  important debrief summary. A separate region gives precise control and lets both be heard.
- **Watched RED** (`test/flow_a11y.cjs` #18a): pre-fix, the CTA label ("Back to the drill") is in no
  live region (only "Walkthrough" from the pane switch). #18b (armed legend silent) carries a per-run
  self-test: a planted `Grade 1 Missed 2 Shaky 3 Solid` in a live region MUST trip the detector, or the
  assertion is declared inert and the check fails.

#### 6. Seg pip is visual-only (audit #19)
- **Root cause:** the pip is a pure `::before` dot; the pipped tab's accessible name is unchanged, so
  AT gets no "recommended next" cue.
- **Fix:** `flowPip()` sets the recommended tab's `aria-describedby` to an off-screen
  `#flowpip-desc` = "Recommended next" (and clears it when the pip moves). A NAME, per #18's
  philosophy — read on focus, not announced live. Off-screen (not `display:none`) so every SR computes
  the description; `position:absolute` keeps it out of flow (zero layout/paint/VR impact). Applied to
  the `.flow-cta` mock/mix markers too, for consistency.
- **Watched RED** (`test/flow_a11y.cjs` #19): pre-fix, the pipped tab's `aria-describedby` is null.

#### 7. `.flow-go` shadow focus ring (audit #20)
- **Root cause:** `.flow-go` (the terminal strip button, base-styles.js:167) lives in a shadow root;
  the document's `button:focus-visible{outline:2px solid var(--acc)}` can't cross the boundary, so Tab
  landed the ~1px UA outline (measured: `1px auto rgb(16,16,16)`).
- **Fix:** `.flow-go:focus-visible{outline:2px solid var(--acc);outline-offset:2px}` in BASE_SHEET
  (reaches all 5 flow-strip shadow scopes).
- **Deviation from the brief's "-2px inward" (flagged):** that recipe is `.mock-body`'s
  (shared-sheets.js:75) — a PLAIN surface, where an inset accent ring is visible. `.flow-go` is a
  SATURATED accent GRADIENT (`linear-gradient(var(--acc),var(--acc2))`); an inset `var(--acc)` ring
  paints accent-on-accent (invisible). OUTWARD (offset 2px) lands the ring on the pane bg — matching
  the SIBLING `.revset-b:focus-visible` in the same drill shadow scope (drill/logic.js:204), so this
  reuses an in-scope recipe rather than inventing one, and produces a VISIBLE ring (the constraint's
  intent).
- **Watched RED** (`test/flow_a11y.cjs` #20): pre-fix, `:focus-visible` outline is `1px auto
  rgb(16,16,16)` — not `var(--acc)` (rgb(0,107,99)) at 2px solid. Measured with
  `.focus({focusVisible:true})` (deterministic `:focus-visible`) + a `var(--acc)`→rgb probe.

## The one item verified and SKIPPED

### Dock / terminal-strip dedupe (audit #11) — VERIFY-FIRST → **SKIP, flagged**

The audit saw the dock and a terminal hand-off strip emit byte-identical copy "~200px apart", but
INDUCED the co-occurrence by mounting a strip on a NON-terminal pane. The brief required establishing
whether they co-occur on a **true** terminal before acting.

**Driven live** (committed deliverable, dark, data-storage drill):
- **All-solid debrief (shk=0):** dock shows "Try the whiteboard →" at **(x19, y230)** — the SIDEBAR;
  the drill's `#dflow` strip shows the same "Try the whiteboard →" at **(x370, y1618)** — the STAGE.
  They co-occur, but ~1400px apart in **different columns**, not the "~200px apart, one column" the
  audit induced (and a 1280×900 viewport can't show both at once).
- **Any revisit (shk>0):** the strip **self-dedupes** entirely (`rec.self = self.shk > 0`,
  drill/logic.js:758) — only the dock shows. No co-occurrence at all.
- **Structural:** the dock is the only sidebar "next" affordance; every strip mounts in a pane/overlay
  (stage). So dock and strip are ALWAYS in different columns — the audit's same-column stack cannot
  occur in the real layout.

**Why SKIP the code:** mutual exclusion ("strip mounted → dock demotes") would require the dock to read
the pane's LIVE DOM ("is a strip mounted here?"), violating the dock's pure-data / perf-deferral
discipline (`nextUp` "reads no pane's live state, so it never trips the perf deferral rule",
session-progress.js) — a net architectural regression, and the exact discipline the D3 perf wave
protects. For a benign, different-column, one-compute-consistent echo that's rarely co-visible, the
cost isn't worth it. **Team-lead accepted this verdict** (the architecture-cost argument is decisive).
